# Летняя физико-математическая школа «Квант» — Стартовый проект сайта

Добро пожаловать в стартовый проект для веб-разработки на летней смене ЛФМШ «Квант»! 

Этот проект представляет собой базовую заготовку сайта нашей школы. В течение смены каждый из вас сможет добавить новые функции: личные кабинеты с достижениями, интерактивное расписание занятий, систему публикации новостей и многое другое. Разработка будет вестись в индивидуальных Git-ветках, которые затем будут объединены.

## Инструкция по запуску

### 1. Клонирование репозитория

Для начала скопируйте проект к себе на компьютер. Откройте терминал (или командную строку) и выполните команды:

```bash
# Клонируем проект с удаленного сервера (замените URL на ваш адрес)
git clone <repository_url>

# Переходим в папку с проектом
cd quant_site
```

### 2. Создание виртуального окружения

Виртуальное окружение — это «изолированная песочница» для проекта. Оно нужно, чтобы библиотеки нашего сайта (например, Django) не конфликтовали с другими проектами на вашем компьютере.

Создайте окружение с именем `.venv` следующей командой:

```bash
python -m venv .venv
```

### 3. Активация окружения

Чтобы компьютер понял, что мы хотим работать именно внутри нашей «песочницы», её нужно активировать.

* **Для Windows (PowerShell):**
  ```powershell
  .venv\Scripts\activate
  ```
  *(Если возникает ошибка безопасности, попробуйте команду в обычной командной строке `cmd`: `.venv\Scripts\activate.bat`)*

* **Для macOS и Linux (Bash / Zsh):**
  ```bash
  source .venv/bin/activate
  ```

После успешной активации в начале строки терминала появится круглая скобка с именем окружения: `(.venv)`.

### 4. Установка зависимостей

Установим Django и необходимые библиотеки, список которых сохранен в файле `requirements.txt`:

```bash
pip install -r requirements.txt
```

### 5. Применение миграций

Миграции — это чертежи для нашей базы данных. Django использует их, чтобы создать таблицы для пользователей, сессий и других системных данных. Выполните команду:

```bash
python manage.py migrate
```
После этого в корне проекта появится файл базы данных `db.sqlite3`.

### 6. Создание учетной записи администратора

Создайте «суперпользователя» (главного администратора), чтобы иметь доступ к панели управления сайтом:

```bash
python manage.py createsuperuser
```
Терминал попросит вас ввести:
1. **Username** (Имя пользователя, например: `admin`).
2. **Email** (Можно оставить пустым и нажать *Enter*).
3. **Password** (Пароль. При вводе символы *не будут отображаться на экране* ради безопасности. Просто введите пароль и нажмите *Enter*, затем повторите его).

### 7. Запуск локального сервера

Теперь все готово к первому запуску! Запустите встроенный сервер Django:

```bash
python manage.py runserver
```

В терминале побегут строчки логов, и в конце вы увидите адрес запуска. 

* Чтобы открыть сайт, перейдите в браузере по адресу:  
  👉 **[http://127.0.0.1:8000/](http://127.0.0.1:8000/)**
* Чтобы зайти в панель администратора, перейдите по адресу:  
  👉 **[http://127.0.0.1:8000/admin/](http://127.0.0.1:8000/admin/)** (используйте логин и пароль администратора, которые создали на шаге 6).

*Для остановки сервера нажмите комбинацию клавиш **Ctrl + C** в окне терминала.*

## Структура проекта

Давайте разберем, за что отвечают ключевые папки и файлы проекта:

* **`quant_site/`** — конфигурационная папка проекта. В ней лежит файл `settings.py` (где настраиваются базы данных, языки и подключаются новые приложения) и файл `urls.py` (главный список адресов сайта).
* **`main/`** — наше основное Django-приложение. Именно здесь мы пишем код обработки страниц. Внутри лежат:
  - `views.py` — функции-представления (логика страниц).
  - `urls.py` — адреса страниц этого приложения.
  - `models.py` — описание таблиц нашей базы данных.
* **`templates/`** — папка с HTML-файлами (шаблонами). Файл `base.html` содержит общий каркас сайта, а файлы в подпапках (например, `templates/main/index.html`) наследуют его структуру и выводят уникальный контент страниц.
* **`static/`** — папка со статическими ресурсами: кастомные файлы стилей `css/style.css`, картинки, шрифты.
* **`manage.py`** — главный скрипт-помощник Django, через который мы запускаем сервер, делаем миграции и управляем проектом.

## Полезные команды Django

В процессе работы над проектом вы будете часто использовать следующие команды:

1. `python manage.py makemigrations` — сканирует файл `models.py` на наличие изменений в коде моделей и создает новые файлы-чертежи (миграции) в папке `migrations/`.
2. `python manage.py migrate` — читает все новые миграции и применяет их к базе данных SQLite, создавая или изменяя таблицы.
3. `python manage.py createsuperuser` — создает учетную запись администратора с полным доступом к панели управления `/admin/`.
4. `python manage.py shell` — открывает интерактивную консоль Python, подключенную к окружению Django. Через нее можно тестировать код, обращаться к базе данных и создавать объекты вручную.
5. `python manage.py collectstatic` — собирает все статические файлы (CSS, картинки) со всех приложений в одну общую папку для работы на реальном сервере (production).
